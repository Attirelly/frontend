      A component for selecting a multi-level product category using cascading dropdowns.

      This component constitutes a step in the product upload form. It fetches a complete hierarchy of categories from the backend and presents the user with a series of dependent dropdowns. Selecting a category at one level populates the options for the next level. The component validates that a complete, four-level category path has been selected before allowing the user to proceed.

      State Management

      • Local State (useState): Manages the complete list of categories fetched from the API, the user's current selections at each level, and the visibility state of each dropdown.
      • Global State (product_upload_store): It reads the initial category state from the store (useFormData) and continuously updates the store with the user's selections (updateFormData). It also updates the step's validation status (setStepValidation).

      API Endpoint

      GET /categories/ This endpoint is called once on component mount to fetch the entire category tree. The component then handles the parent-child relationships and filtering on the client side.

      • Returns: Category[] - A flat array of all category objects.
